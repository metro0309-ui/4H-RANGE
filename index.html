가격 위치
Low High
① 백테스트 엔진 완전 재설계 핵심
기존: 다음 3봉 High가 TP 위크만 스쳐도 Win → 과낙관
개선:
Step 1: 매수가(Low+buf)에 LOW가 실제 닿아야 체결
Step 2: 체결 후 캔들마다 Stop vs TP 순서 체크
Step 3: 같은 캔들에서 둘 다 닿으면 → 보수적으로 LOSS
Step 4: 최대 6봉(24시간) 내 미결 = 타임아웃 제외
수수료+슬리피지 0.15% 차감 후 순수익 계산
② ATR14 동적 Stop 변동성 반영
TrueRange = max(H−L, |H−Cprev|, |L−Cprev|)
ATR14 = EMA of TrueRange over 14 periods
Stop = BuyPrice − (ATR14 × 1.5) // 동적
고정 Range×5% 대신 실제 변동성 기반. BTC/고변동 코인은 Stop이 더 넓게, 저변동 코인은 좁게 자동 적용
③ Volume 필터 False Breakout 방지
avgVol20 = 20봉 평균 거래량
진입 조건: 신호봉 Volume ≥ avgVol20 × 0.8
저거래량 구간의 Range는 신뢰도 낮음. 거래량 충분한 봉의 Range만 백테스트에 포함
④ Close 확인 로직 Wick false signal 방지
신호 유효 조건: Close ≥ Low + Range × 0.20
신호봉이 하단 20% 이하에서 종가 마감 = 하락 모멘텀. 이 경우 Range를 신뢰할 수 없으므로 해당 봉 백테스트 제외
⑤ Daily MTF Bias 추세 필터
Daily 20MA = 20일 종가 단순이동평균
강세 = Daily Close > Daily 20MA
백테: 강세 구간 진입만 카운트
일봉 추세 역행 매수는 체결 후에도 Stop까지 밀릴 확률이 높음. Daily API 데이터 병렬 fetch
최적 버퍼 선택 기준 복합점수
Score = WinRate × ProfitFactor × ExpectedValue
단순 승률 최고값이 아닌, 수익성(PF)과 기대값(EV)까지 복합 평가. 승률 60%라도 PF가 낮으면 탈락
